# 《后晋》游戏开发 - 完整优化测试场景

## 使用说明
将下面的 prompt 分段输入到 s_full.py 的 REPL 中，观察团队协作效果。

---

## 阶段一：团队初始化（输入这段）

你好，我是项目总监。现在启动《后晋》游戏开发项目。

请执行：
1. 创建队友"刘裕"，角色"技术Leader"，任务"你是技术Leader，负责项目规划。先用 task 工具（subagent）探索当前 agents 目录下的代码结构，了解项目情况，然后等待我的下一步指示。"

2. 创建队友"刘无忌"，角色"Python开发工程师"，任务"你是开发工程师刘无忌。等待任务看板上出现开发任务，自动认领并完成。开发方式：创建对应文件写入'模拟代码'，然后用 background_run 执行 'sleep 10 && echo done'，完成后发消息给张牧之通知测试，并将任务标记为completed。"

3. 创建队友"张牧之"，角色"测试工程师"，任务"你是测试工程师张牧之。等待任务看板上出现测试任务，自动认领。测试方式：在对应文件追加'测试通过'，用 background_run 执行 'sleep 5 && echo test_done'，有30%概率发现bug（随机），如发现bug则创建修复任务并通知刘无忌，否则标记completed。"

4. 列出所有队友确认创建成功

用 TodoWrite 跟踪这4个步骤。

---

## 阶段二：任务拆分（等阶段一完成后输入）

刘裕已经完成探索，现在请：

1. 给刘裕发消息："请将《后晋》游戏拆分为10个开发任务，要求有依赖关系。格式：
   - 任务1: 游戏架构设计（无依赖）
   - 任务2: 核心引擎（依赖任务1）
   - 任务3-10: 各功能模块（根据逻辑设置依赖）
   
   每个任务用 task_create 创建，subject 格式'[开发]功能名'，有依赖的用 task_update 设置 add_blocked_by。全部创建完成后用 broadcast 通知团队'任务已发布，请查看看板'。"

2. 等待30秒让刘裕处理

3. 用 task_list 查看任务看板

用 TodoWrite 跟踪进度。

---

## 阶段三：开发和测试循环（等阶段二完成后输入）

现在任务已发布，刘无忌和张牧之会自动工作。

请你作为项目总监：

1. 每隔20秒检查一次：
   - 用 task_list 查看任务进度
   - 用 read_inbox 查看消息
   - 用 list_teammates 查看成员状态

2. 当完成3个任务时，用 broadcast 发送："进度不错，继续加油！"

3. 当完成6个任务时，用 broadcast 发送："已完成过半，保持节奏！"

4. 重复检查直到所有任务 completed

用 TodoWrite 跟踪检查次数和广播次数。

---

## 阶段四：技术报告和收尾（等所有任务完成后输入）

所有开发和测试任务已完成。

请：
1. 给刘裕发消息："请生成技术报告，内容包括：完成的功能模块、遇到的问题、团队协作情况。报告写入文件 test_output/tech_report.md"

2. 等待刘裕完成报告

3. 读取报告内容

4. 用 broadcast 通知："项目第一阶段完成，感谢大家！"

5. 依次用 shutdown_request 关闭：刘无忌、张牧之、刘裕

6. 用 list_teammates 确认所有成员状态

用 TodoWrite 跟踪这6个步骤。

---

## 阶段五：验证和总结（等所有成员 shutdown 后输入）

项目完成，请验证：

1. 用 bash 执行 "ls -la test_output/" 查看生成的文件

2. 用 bash 执行 "find .tasks -name '*.json' | wc -l" 统计任务数

3. 用 bash 执行 "cat .team/config.json" 查看团队配置

4. 用 read_file 读取 "test_output/tech_report.md" 

5. 总结本次测试覆盖的功能点

---

## 预期测试覆盖

✅ spawn_teammate - 创建3个不同角色成员
✅ task (subagent) - 刘裕用子代理探索代码
✅ task_create - 创建10个游戏开发任务
✅ task_update - 设置任务依赖关系（blockedBy）
✅ task_list - 多次查看任务看板
✅ 自动认领 - 成员自动claim无依赖任务
✅ background_run - 异步执行开发和测试
✅ send_message - 成员间点对点通信
✅ broadcast - Leader群发消息
✅ read_inbox - 接收和处理消息
✅ list_teammates - 查看团队状态
✅ shutdown_request - 优雅关闭成员
✅ write_file - 创建代码和报告文件
✅ bash - 验证文件系统状态
✅ TodoWrite - 跟踪每个阶段进度

---

## 时间估算

- 阶段一：2-3分钟（创建成员+探索）
- 阶段二：1-2分钟（拆分任务）
- 阶段三：5-10分钟（开发测试循环，取决于任务数和sleep时间）
- 阶段四：1-2分钟（报告和关闭）
- 阶段五：1分钟（验证）

总计：10-18分钟

## 可选的进阶测试

如果想测试更多功能，可以在阶段三插入：

**测试bug修复流程：**
"张牧之，在测试任务3时，请发现一个bug并创建修复任务'[修复]任务3的登录问题'，然后发消息给刘无忌。"

**测试压缩机制：**
在对话轮次较多时，手动执行 "compress" 工具或输入 "/compact" 命令。

**测试技能系统：**
先创建 skills/game_dev/SKILL.md 文件，然后让刘无忌 load_skill。
